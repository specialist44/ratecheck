Notlar & sık sorulanlar

Git ve GitHub farkı ne?

Git: bilgisayarındaki proje klasörünün zaman makinesi. GitHub: Git kayıtlarının internete yedeklendiği, Vercel'in okuduğu site. Her şey Claude Code üzerinden, bilgisayarda yapılacak.

Docker kullanacak mıyız?

Hayır. RateCheck tamamen frontend bir proje, Vercel kodu doğrudan GitHub'dan okuyup yayınlıyor. Docker hiçbir aşamada gerekmiyor.

Claude Code izin modları (Shift+Tab)

Default: her değişiklikte sorar. Accept edits: dosya değişikliklerini otomatik onaylar, riskli komutlarda yine sorar — kullanılan mod bu. Tam "hiç sorma" modu önerilmiyor.

"Caveman modu" nedir

Claude'un açıklama metinlerini kısaltan üçüncü parti eklenti — kod kalitesini etkilemez. Kurulum: /plugin marketplace add JuliusBrussee/caveman → /plugin install caveman@caveman → /reload-plugins

Effort / Thinking ayarı

Basit adımlarda düşük effort + kapalı thinking yeterli, hız kazandırır. Kod/mantık içeren adımlarda yüksek effort önerilir.

Kullanım hakkı biterse ne olur?

"Usage limit reached, resets at [saat]" mesajı çıkar. Dosyalara zarar gelmez. Belirtilen saatte devam et — memory-bank.md sayesinde kaldığın yerden sürer.

Tuhaf/bozuk bir sonuç görürsen önce ne yap?

Kod değişikliği sonrası beklenmedik bir şey görürsen önce Ctrl+Shift+R (sert yenileme) dene — çoğu zaman tarayıcı önbelleğinden kaynaklanır.

VS Code "Restricted Mode" uyarısı çıkarsa?

VS Code yeni açılan klasörlere güvenmediği için bazı özellikleri (Source Control dahil) kısıtlar. Manage → Trust ile kendi projene güvendiğini onaylaman yeterli, zararsız bir adım.

Her önemli değişiklikten sonra ne yapmalı?

git add . ve git commit -m "..." ile o anki çalışan hali kaydet. Bu bir "kayıt noktası" — bir şey bozulursa oraya geri dönülebilir. Küçük metin değişikliklerinde şart değil, ama dosya taşıma/bölme gibi riskli işlemlerden önce ve sonra mutlaka yapılmalı.

LF will be replaced by CRLF uyarısı hata mı?

Hayır. Windows ve Git'in satır sonu karakteri farkından kaynaklanan zararsız bir bilgilendirme, göz ardı edilebilir.

Vercel'de hangi planı seçmeliyim?

"I'm working on personal projects" (Hobby) seç — ücretsiz ve süresiz. "Commercial projects" (Pro) seçeneği $20 kredili ücretli deneme başlatıyor, yanlışlıkla seçilmemeli.

href="#" linkine tıklayınca neden sayfa başına gidiyor?

Bu normal bir tarayıcı davranışı, hata değil — # boş bir bağlantı yer tutucusudur (placeholder). Gerçek bir URL girildiğinde bu davranış otomatik olarak düzelir.

Neden noindex kaldırma işini domain almadan önce yapmıyoruz?

Şimdi kaldırırsak Google, adresi geçici Vercel adresini (ratecheck-ebon.vercel.app) indexler; domain alınca da yeni adrese geçmesi gerekir. Bu gereksiz bir ara adım ve küçük bir kafa karışıklığı yaratır — o yüzden noindex kaldırma, domain bağlandıktan sonraya bırakıldı.
